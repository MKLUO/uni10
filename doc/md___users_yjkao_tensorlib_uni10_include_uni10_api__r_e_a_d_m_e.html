Uni10-v2.0 API Changelogs

Why the usage of Uni10-v2.0 differ from Uni10-v1.0 ?

To make the source codes of Uni10 more formalism, maintainable and extendable. In the refactoring we made three major decisions.

  1. Classes Matrix and UniTensor are changed to Class Templates.
  2. Linear-solvers are moved out of classes Matrix and UniTensor.
  3. Clean up source codes by following Google C++ Style

Hence, taking a long-term view, the modifications of Uni10 usage is necessary.

In the following sections, we expose the changelogs more detail.

2. Function Names

Following the function naming rules of Google C++ Style Guide,

  1. Ordinary functions should start with a capital letter and have a capital letter for each new word.
  2. Accessors and mutators (get and set functions) may be named like variables
